# Задание 1

Необходимо написать функцию для валидации постов юзеров.

Требования:
1) Пользователь в сообщениях может использовать следующие HTML теги и атрибуты:
<a href="" title=""> </a>
<code> </code>
<i> </i>
<strike> </strike>
<strong> </strong>
2) Проверка на наличие тегов и атрибутов, не указанных в пункте 1, а также их некорректных комбинаций. В таком случае пост считается невалидным.
3) Проверка на наличие закрывающих тегов.
4) Проверка на корректную вложенность тегов.
5) Проверка на валидность XHTML.
6) В решении должно использоваться хотя бы одно регулярное выражение.
7) Запрещено использовать встроенные функции DOM и подобные.
8) Реализация через одно регулярное выражение без дополнительной логики не принимается.
9) Результатом должна быть функция function validatePost($post), которая возвращает true, если сообщение валидно, и false в обратном случае.


# Задание 2

Необходимо реализовать ограничение частоты отправки комментариев пользователями.
Требования:
1) Для каждого пользователя (userId) необходимо ограничить количество комментариев: не более 3 комментариев за период 10 секунд
2) Текущая версия решения использует Redis для хранения состояния лимитера.
3) Состояние сохраняется между вызовами метода и может использоваться в долгоживущем процессе.
4) Интерфейс класса:

class CommentLimiter
{
    public function canPost(int $userId): bool;
}
